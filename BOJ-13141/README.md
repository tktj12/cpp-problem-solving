# Problem
* https://www.acmicpc.net/problem/13141
* 빠르게 그래프 태우기
* 난이도 : 플레티넘 5

# Solution

### 방법 1 : Dijkstra, brute force
* 풀이
  * 결국 모든 간선들을 태운다는 것은 모든 간선들을 거쳐야 한다는 의미이다. 따라서 간선마다 간선의 중간 지점에 새로 정점을 하나 만든다. 
  다익스트라 알고리즘은 모든 정점에 도달하는 시간을 알아낼 수 있다. 따라서 그래프를 모두 태우는데 얼마나 걸리는지도 알아낼 수 있다.
  * 이런식으로 정점을 생성하면 간선이 최대 40000개가 되고 정점은 20200개가 되지만 다익스트라 알고리즘은 최대 200개의 정점에서만 돌리면 되기 
  때문에 시간 복잡도가 생각보다 커지지 않아서 시간 안에 동작한다.

* 시간 복잡도 :
  * $O(NE\log E)$
<br></br>

### 방법 2 : Floyd-Warshall, brute force
* 풀이
  * 플로이드로 간선 사이 최단 거리를 전부 구해준 뒤, 모든 정점에 불을 붙여보는 것으로 해결할 수 있다. 
각 정점 사이의 최장 간선을 따로 테이블로 관리하여, 그 간선이 타는데 걸리는 시간 중 최댓값이 그래프 전체가 타는데 걸리는 시간과 같다.

* 시간 복잡도 :
  * $O(N^3)$
<br></br>
