# Problem
* https://www.acmicpc.net/problem/19585
* 팀명이 미리 주어진 특정 색상과 닉네임의 결합인지 여러번 판별하기
* 난이도 : 플레티넘 3

# Solution

### 방법 1 : Trie
* 풀이
  1. 색상은 Trie에 그대로 넣고 닉네임은 Trie에 거꾸로 넣는다. 즉, 닉네임 nick을 Trie에 넣을 때
k->c->i->n 순으로 넣는다.
  2. 쿼리로 주어진 팀명은을 볼 때, 먼저 앞에서부터 탐색한다. 이때 특정 색상의 이름과 일치할 때마다 해당 인덱스를 표시한다. 
  트라이의 terminal 멤버 변수를 이용해 손쉽게 알 수 있다.
  3. 이번엔 팀명을 뒤에서부터 탐색한다. 특정 닉네임과 일치할 때마다 2번에서 마킹해 놓은 표시가 있는지 확인한다. 
  이렇게 하면 팀명을 최대 두 번 탐색하여 답을 구할 수 있다.
  * 자식 노드를 가리키는 배열의 메모리를 아끼기 위해 map을 사용했다. 하지만 map자체도 무거워서 큰 효과는 없었다.

* 시간 복잡도 :
  * $O(N)$
  * $N$ : 입력으로 주어지는 문자열의 총 길이
