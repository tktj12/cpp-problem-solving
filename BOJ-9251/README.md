# Problem
* https://www.acmicpc.net/problem/9251
* 두 수열의 공통 부분 수열의 최대 길이
* 난이도 : 골드 5

# Solution

### 방법 1 : 이분 탐색
* 풀이
  * LIS의 $O(n\log n)$ 풀이법처럼 풀었다.
  * $end_idx$의 i번째 원소 : 길이가 $i+1$인 공통 부분 수열이 끝나는 가장 빠른 인덱스. 
  즉, 문자열 "ABCBC"와 "BC"가 있을 때 길이가 1인 공통 부분 수열인 "B"와 "C"가 끝나는 인덱스는 1,2,3,4가 있는데 
  그 중 가장 빨리 끝나는 1을 $end_idx$는 저장하고 있다. 그리고 같은 원리로 길이가 2인 공통 부분 수열이 끝나는 인덱스는 
  2로 저장하고 있다. 이렇게 데이터를 저장해놓으면 최장 공통 부분 수열을 구하기 용이하기 때문이다.   
  문자열 "BC" 뒤에 문자 'B' 하나가 더 있다고 가정해보자. 문자열 "BC" 까지 찾은 LCS는 길이가 2이고 인덱스 2에서 끝난다. 
  그런데 "ABCBC"의 인덱스 3에 'B'가 있으므로 LCS는 3으로 늘어나고 길이가 3인 CS의 최소 인덱스는 3이된다.
  * 현재 탐색하는 문자 X가 나타나는 모든 인덱스 i에 대해서 가장 큰 i부터 위의 과정대로 수행하여 항상 공통 부분 수열에 길이에 따라 
  최소 인덱스를 최신화 시켜준다. 가장 작은 i부터 하게되면 X가 나타나는 횟수만큼 LCS의 길이가 증가할 수 있기 때문에 잘못된 방법이다. 
  최소 인덱스의 위치는 이분 탐색(map)을 이용해서 빠르게 접근한다.

* 시간 복잡도 :
  * $O(n^2\log n)$
