# Problem
* https://algospot.com/judge/problem/read/PI
* 부분적인 숫자 패턴을 찾고, 최적의 경우를 계산하는 문제
* 난이도 : 하

# Solution

### 방법 1 : 동적 계획법
* 풀이
  * 입력의 i번째부터 끝까지의 최소 난이도를 계산하는 것을 부분문제로 생각하고 재귀함수를 구현
  * i값을 기준으로 메모이제이션을 구현

* 시간 복잡도 :
  * $O(N)$
<br></br>

# Learning
 * `switch (n) {}` 이같은 스위치문을 사용할 때, 스위치문 중간에 n의 값이 바뀌는 것에 주의해야한다.   
스위치문은 `case` 예약어마다 n을 계산하기 때문이다.
