# Problem
* https://algospot.com/judge/problem/read/SUSHI
* 가격과 선호도가 다른 음식들을 주어진 예산 내에 구매하여 최대의 선호도 구하기
* 난이도 : 상

# Solution
* 그리디로는 안 됨.   
예산이 10000원, 음식1 : 5100원, 선호도 199  / 음식2 : 5000원, 선호도 100인 경우 음식2를 두 개 먹는 게 이득이다.
* 완전 탐색은 시간이 너무 오래 걸림. 예산의 최대가 $2^31$이나 된다.   
완전탐색을 한다면 i번 음식을 0개부터 최대까지 선택하는 경우를 전부 탐색해봐야 한다.
* 한 음식보다 가성비가 나쁜데 더 비싼 음식은 필요없음.

### 방법 1 : 완전 탐색
* 풀이
  * i번 음식을 0번 부터 최대한 사는 경우를 모두 고려

* 시간 복잡도 :
  * $O(m^2n)$
<br></br>

### 방법 2 : 반복적 동적 계획법
* 풀이
  * &budget$만큼의 예산을 갖고 $i$번째 음식까지의 선호도의 최선
$best(budget, i)$는 현재 $i$번째 음식을 먹지 않거나, $price_i$만큼 값을 지불하고 $i$번째 음식을 하나 더 먹는
경우 중 최선의 경우 $max(best(budget, i-1),best(budget-price_i, i) + value_i)$ 이다. ($value_i$ : 선호도, 코드랑 다름)
  * 이것을 바탕으로 반복문를 사용한 동적 계획법으로 해결한다.

* 시간 복잡도 :
  * $O(mn)$
<br></br>

오름차순 정렬을 하고 싶으면 '>' 연산자를 정의하고 `sort()` 함수의 세 번째 인자로 `greater<>()`를 주면 된다.
